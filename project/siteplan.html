Personal Portfolio Website – Site Plan

This planning document outlines structure, content, and design for my personal portfolio.

Site Name

Daniel Ayodele Portfolio

This name clearly identifies the site as my professional portfolio to support job applications and freelance inquiries. Logo: a simple monogram “DA” mark will be used in the header and favicon.

Optional domain to consider later: ayodele-dev.com

Site Purpose

The site serves as a hub to showcase my projects, skills, and background. It will provide:

  • Featured projects with tech stacks and links to GitHub/live demos
  • Skills overview and short bio (About)
  • Accessible contact form and social links

Scenarios

  • “Where can I quickly view Daniel’s most relevant projects and the technologies used?”
  • “How do I contact Daniel to request freelance work or an interview?”

Typography

  • Headings: System UI stack (Segoe UI/Helvetica/Arial) – clear and modern.
  • Body: Same stack for consistency and performance.

Using a system font stack avoids external requests for this planning doc. The final site may adopt a web font if needed.

Wireframes

Mobile Layout

  • Header with site title and menu toggle
  • Hero: photo + tagline
  • About section
  • Projects as stacked cards
  • Skills list
  • Contact form
  • Footer with social links and references link

Desktop Layout

  • Projects grid (3–4 cards)
  • Skills grid with icons
  • Contact form full width
  • Footer across the bottom
