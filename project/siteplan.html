Personal Portfolio Website – Site Plan

Site Name

Daniel Ayodele Portfolio

I chose this name because it represents my personal brand and helps visitors quickly understand that this website showcases my skills, projects, and professional background as a developer.

Site Purpose

The purpose of this portfolio website is to present my work, skills, and background to potential employers, clients, and collaborators. The site will include project showcases, contact information, technical skills, and professional information. It serves as a digital resume and portfolio to support job applications or freelance opportunities.

Scenarios

  • A recruiter wants to see examples of my past projects and the technologies I have used.
  • A potential client wants to contact me for a freelance job and needs an easy way to send a message or request.

Color Scheme

These are the primary colors I will use across the website.

Primary Color – #1a73e8

Used for headings, navigation links, and call-to-action buttons.

Accent Color – #fbbc05

Used for highlights, icons, section dividers, and accents.

Typography

Headings: Poppins – clean, modern, ideal for titles and emphasis.

Body Text: Roboto – simple, readable, good for paragraphs.

These two fonts work together for a professional and accessible layout.

Wireframes

Mobile Layout (Example)

A stacked vertical layout with:

  • Header + Mobile Menu Toggle
  • Hero section with photo + tagline
  • About section
  • Projects displayed in cards
  • Skills list
  • Contact form
  • Footer with social links

Desktop Layout (Example)

A two‑column or grid layout with:

  • Top navigation bar
  • Hero section split: image on left, intro text on right
  • 3–4 project cards in a responsive grid
  • Skills grid with icons
  • Contact form with full‑width layout
  • Footer across the bottom

(Visual wireframe diagrams will be drawn for the final project pages.)
